**2. CREATE STATE MACHINE STRUCTURE**

Inside the system assembly part def, add:
```sysml
state def SystemStateMachine;

    entry; then INITIAL_STATE_NAME;

```

**3. MAP EACH STATE FROM JSON**

For each state in the JSON "states" array:

```json
{
  "name": "STANDBY",
  "type": "normal",
  "description": "Monitoring active...",
  "hasEntryActions": true,
  "entry_actions": ["InitializeControllers", "VerifyBatteryStatus"],
  "hasExitActions": false,
  "exit_actions": [],
  "hasDoActions": true,
  "do_actions": ["MonitorSystemStatus"],
  "hasInvariants": true,
  "invariants": [
    {
      "parameter": "battery.soc",
      "condition": "greater_than",
      "value": { "value": 0.05, "unit": "" }
    }
  ]
}
```

BECOMES:

```sysml
state STANDBY {
    // Entry Actions: InitializeControllers, VerifyBatteryStatus
    // Do Action: MonitorSystemStatus
    // Invariant: battery.soc > 0.05
}
```

**IMPORTANT:** 
- Use the state "name" field for the SysML state name
- Put actions in comments (SysML v2.0 state actions are still evolving)
- Invariants can be mentioned in comments or as assert constraint blocks if critical

**4. MAP EACH TRANSITION FROM JSON**

For each transition in the JSON "transitions" array:

```json
{
  "from": "STANDBY",
  "to": "RUNNING",
  "hasTrigger": true,
  "trigger": { "type": "event", "name": "StartCommand" },
  "hasGuards": true,
  "guards": [
    {
      "parameter": "battery.soc",
      "operator": "greater_than",
      "value": { "value": 0.1, "unit": "" }
    }
  ]
}
```

BECOMES (placed after the "from" state definition):

```sysml
accept StartCommand if battery.soc > 0.1 then RUNNING;
```

**Mapping:**
- trigger.name → accept clause
- guards[0].parameter, operator, value → if clause
- "to" → then clause

**Guard Operator Mapping:**
- "greater_than" → >
- "less_than" → <
- "equals" → ==
- "greater_than_or_equal" → >=
- "less_than_or_equal" → <=

**5. HANDLE STATES WITHOUT GUARDS**

If hasGuards is false or guards array is empty:

```json
{
  "from": "OFF",
  "to": "STANDBY",
  "hasTrigger": true,
  "trigger": { "type": "event", "name": "PowerOn" },
  "hasGuards": false
}
```

BECOMES:

```sysml
accept PowerOn then STANDBY;
```

**6. HANDLE MULTIPLE GUARDS**

If there are multiple guards in the guards array, combine them with "and":

```json
{
  "guards": [
    { "parameter": "battery.soc", "operator": "greater_than", "value": { "value": 0.1, "unit": "" } },
    { "parameter": "temperature", "operator": "less_than", "value": { "value": 323, "unit": "K" } }
  ]
}
```

BECOMES:

```sysml
accept StartCommand if battery.soc > 0.1 and temperature < 323 [K] then RUNNING;
```

**7. HANDLE STATE INVARIANTS (OPTIONAL)**

If a state has invariants and they are marked as critical, you can add them as assert constraint blocks:

```json
{
  "invariants": [
    {
      "parameter": "battery.soc",
      "condition": "greater_than",
      "value": { "value": 0.05, "unit": "" },
      "critical": true
    }
  ]
}
```

COULD BECOME (inside the state):

```sysml
state STANDBY {
    // Entry Actions: InitializeControllers, VerifyBatteryStatus
    // Do Action: MonitorSystemStatus
    
    assert constraint {
        battery.soc > 0.05
    } doc /* Critical: Minimum battery charge required */
}
```

**NOTE:** For simplicity, you can put invariants in comments if preferred.

**8. STATE ORDERING AND INITIAL STATE**

The state with "type": "initial" should be the first state in entry; then:

```sysml
state systemStates : SystemStateMachine {
    entry; then OFF;  // ← OFF is the initial state
    
    state OFF { ... }
    // ...
}
```
